# Onboarding
You now have access to SpecPowers! Introduce the workflow briefly and let the user know they can start by saying "I want to build a feature" to invoke the `exploring` skill.

# When to Load Steering Files
- For session bootstrap and skill routing → `skills/using-skills/SKILL.md`
- When exploring requirements for a new feature → `skills/exploring/SKILL.md`
- When proposing high-level scope and intent → `skills/proposing/SKILL.md`
- When specifying behavior with GIVEN/WHEN/THEN → `skills/specifying/SKILL.md`
- When doing architecture design → `skills/designing/SKILL.md`
- When breaking down into TDD tasks → `skills/planning/SKILL.md`
- When executing tasks and enforcing Spec compliance → `skills/spec-driven-development/SKILL.md`
- When finalizing Delta Spec merging and change history → `skills/archiving/SKILL.md`
- When enforcing RED-GREEN-REFACTOR iron law → `skills/test-driven-development/SKILL.md`
- When debugging issues systematically → `skills/systematic-debugging/SKILL.md`
- When dispatching a code reviewer subagent → `skills/requesting-code-review/SKILL.md`
- When handling code review feedback  → `skills/receiving-code-review/SKILL.md`
- When forwarding evidence before claims (Verification) → `skills/verification-before-completion/SKILL.md`
- When creating or tuning new skills → `skills/writing-skills/SKILL.md`
