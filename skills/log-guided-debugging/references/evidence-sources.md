# 证据来源

运行时证据可能来自本地文件，也可能来自外部系统。先发现当前仓库和当前环境实际提供什么。

常见来源包括：

- 本地日志
- 云日志
- Alibaba Cloud SLS / Log Service
- MCP 工具
- traces
- metrics
- APM
- 错误上报
- 仪表盘
- 请求样本
- 配置状态
- 部署历史
- 数据状态
- 浏览器/客户端证据
- runbooks

不要假设某个来源一定存在。查找仓库文档、runbooks、已配置的 MCP servers、可用 MCP tools、观测调用点、dashboard 引用、告警名、部署元数据，以及问题里给出的 ID。

如果外部证据不可用，明确说明缺失的证据，然后继续做代码路径分析和目标化观测建议。
